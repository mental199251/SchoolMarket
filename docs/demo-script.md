# 答辩演示脚本

本文档用于 M8 阶段的完整演示走查。默认演示账号由 `python scripts/seed_data.py` 初始化。

## 1. 演示账号

| 账号 | 密码 | 角色 | 用途 |
| --- | --- | --- | --- |
| `user_a` | `Password123` | 普通用户 | 卖家 |
| `user_b` | `Password123` | 普通用户 | 买家 |
| `admin` | `Admin12345` | 管理员 | 后台治理与统计 |

## 2. 启动检查

1. 启动 MongoDB。
2. 启动 Ollama，并在需要 AI 时确认已执行 `ollama signin`。
3. 启动 Flask：`cd server && python run.py`。
4. 启动 H5：`cd frontend && npm run dev:h5`，或在微信开发者工具导入小程序构建目录。
5. 打开 `/health` 和 `/ready`，确认服务与数据库就绪。

## 3. 主流程演示

1. 使用 `user_a / Password123` 登录。
2. 进入发布商品页，填写分类、成色、价格和描述。
3. 点击“生成标题”，采用一个 AI 候选标题。
4. 点击“生成描述”，采用一个 AI 候选描述。
5. 发布商品，进入商品详情页确认标题、价格、状态和描述。
6. 退出或切换账号，使用 `user_b / Password123` 登录。
7. 在商品列表搜索刚发布的商品。
8. 进入详情页，点击“发起购买”。
9. 切回 `user_a`，进入“收到的请求”，确认购买请求。
10. 切回 `user_b`，进入“我的购买”，标记线下交易已完成。
11. 回到商品详情，确认商品状态变为“已售”。

## 4. 管理端演示

1. 使用 `admin / Admin12345` 登录。
2. 查看用户管理，禁用或恢复测试用户。
3. 查看商品管理，对违规商品下架。
4. 发布公告，并在首页确认公告展示。
5. 查看操作日志，确认管理员操作可追踪。
6. 查看统计概览，切换近 7 天、30 天和 90 天。

## 5. 异常路径演示

1. 普通用户访问后台页面，应提示无权限。
2. 对已售商品重复发起购买，应返回状态冲突。
3. 重复点击交易确认或完成按钮，页面只保留一次进行中的操作。
4. 停止 Ollama 后点击 AI 生成，应提示 AI 不可用，原表单内容保留。
5. 停止 MongoDB 后访问 `/ready`，应返回数据库不可用。

## 6. 收尾检查

1. 运行后端测试：`cd server && pytest`。
2. 构建 H5：`cd frontend && npm run build:h5`。
3. 构建微信小程序：`cd frontend && npm run build:mp-weixin`。
4. 备份演示数据库：`cd server && python scripts/backup_database.py`。
